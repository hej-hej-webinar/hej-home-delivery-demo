# Segment A runbook, old world to new world bridge

## Goal
Bridge classic SDLC habits to modern AI-assisted delivery, setting up why controls must evolve without losing delivery confidence.

## Presenter flow
1. Open with the old world pattern, feature first, security later, manual review as the main gate.
2. Introduce the new world pattern, AI-assisted velocity, faster merges, broader change surfaces.
3. Position this demo as the bridge story, same delivery pressure, new risk exposure.
4. Signpost that this segment is mostly narrative, with little to no live coding.

## Talking track
- Teams used to rely on reviewer intuition plus green CI.
- AI acceleration increases output, but it can also multiply hidden risk.
- The answer is not slowing down, it is upgrading governance, policy, and automation.
- The next segments show concrete examples in this baseline repo.
