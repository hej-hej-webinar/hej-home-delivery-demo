# Segment D runbook, Copilot used well and governed

## Goal
Demonstrate high-value Copilot use with governance guardrails, rather than unconstrained generation.

## Presenter flow
1. Start in plan mode and frame the security outcome before touching implementation.
2. Apply adversarial challenge prompts to test assumptions and expose weak spots.
3. Reuse vetted agents or prompt assets for recurring security tasks.
4. Explain governed plugin distribution and approval boundaries for enterprise usage.
5. Reinforce that Copilot quality depends on operator intent, constraints, and review discipline.

## Talking points
- Plan mode reduces reactive coding and improves control traceability.
- Adversarial challenge turns Copilot into a stress test partner.
- Reusable agents capture good practice and reduce variance across teams.
- Governance for plugins and extensions is required to prevent toolchain drift.
